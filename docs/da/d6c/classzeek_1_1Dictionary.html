Detailed Description

A dictionary type that uses clustered hashing, a variation of Robinhood/Open Addressing hashing. The following posts help to understand the implementation:

  • https://jasonlue.github.io/algo/2019/08/20/clustered-hashing.html
  • https://jasonlue.github.io/algo/2019/08/27/clustered-hashing-basic-operations.html
  • https://jasonlue.github.io/algo/2019/09/03/clustered-hashing-incremental-resize.html
  • https://jasonlue.github.io/algo/2019/09/10/clustered-hashing-modify-on-iteration.html

The dictionary is effectively a hashmap from hashed keys to values. The dictionary owns the keys but not the values. The dictionary size will be bounded at around 100K. 1M entries is the absolute limit. Only Connections use that many entries, and that is rare.

◆ Remap() [1/2]

void zeek::Dictionary::Remap ( )
private

One round of remap.

since remap should be very fast. take more at a time. delay Remap when cookie is there. hard to handle cookie iteration while size changes. remap from bottom up. remap creates two parts of the dict: [0,remap_end] (remap_end, ...]. the former is mixed with old/new entries; the latter contains all new entries.
