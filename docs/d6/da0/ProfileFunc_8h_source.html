3// Classes for traversing functions and their body ASTs to build up profiles
4// of the various elements (types, globals, locals, lambdas, etc.) that appear.
5// These profiles enable script optimization to make decisions regarding
6// compilability and how to efficiently provide run-time components.
7// For all of the following, we use the term "function" to refer to a single
8// ScriptFunc/body pair, so an event handler or hook with multiple bodies
9// is treated as multiple distinct "function"'s.
10//
11// One key element of constructing profiles concerns computing hashes over
12// both the Zeek scripting types present in the functions, and over entire
13// functions (which means computing hashes over each of the function's
14// components). Hashes need to be (1) distinct (collision-free in practice)
15// and (2) deterministic (across Zeek invocations, the same components always
16// map to the same hashes). We need these properties because we use hashes
17// to robustly identify identical instances of the same function, for example
18// so we can recognize that an instance of the function definition seen in
19// a script matches a previously compiled function body, so we can safely
20// replace the function's AST with the compiled version).
21//
22// We profile functions collectively (via the ProfileFuncs class), rather
23// than in isolation, because doing so (1) allows us to share expensive
24// profiling steps (in particular, computing the hashes of types, as some
25// of the Zeek script records get huge, and occur frequently), and (2) enables
26// us to develop a global picture of all of the components germane to a set
27// of functions. The global profile is built up in terms of individual
28// profiles (via the ProfileFunc class), which identify each function's
29// basic components, and then using these as starting points to build out
30// the global profile and compute the hashes of functions and types.

44// The type used to represent hashes. We use the mnemonic "p_hash" as

68inline p_hash_type merge_p_hashes(p_hash_type h1, p_hash_type h2)
69 {
70 // Taken from Boost. See for example
71 // https://www.boost.org/doc/libs/1_35_0/doc/html/boost/hash_combine_id241013.html
72 // or
73 // https://stackoverflow.com/questions/4948780/magic-number-in-boosthash-combine
74 return h1 ^ (h2 + 0x9e3779b9 + (h1 << 6) + (h1 >> 2));
75 }
