3 // BaseList.h --
4 // Interface for class BaseList, current implementation is as an
5 // array of ent's. This implementation was chosen to optimize
6 // getting to the ent's rather than inserting and deleting.
7 // Also pairs of append's and get's act like push's and pop's
8 // and are very efficient. The only really expensive operations
9 // are inserting (but not appending), which requires pushing every
10 // element up, and resizing the list, which involves getting new space
11 // and moving the data. Resizing occurs automatically when inserting
12 // more elements than the list can currently hold. Automatic
13 // resizing is done by growing by GROWTH_FACTOR at a time and
14 // always increases the size of the list. Resizing to zero
15 // (or to less than the current value of num_entries)
16 // will decrease the size of the list to the current number of
17 // elements. Resize returns the new max_entries.
18 //
19 // Entries must be either a pointer to the data or nonzero data with
20 // sizeof(data) <= sizeof(void*).

307  // reasons to maybe not refactor to use std::vector ?
308  //
309  // - Harder to use a custom growth factor. Also, the growth
310  // factor would be implementation-specific, taking some control over
311  // performance out of our hands.
312  //
