78/* ---------------------------------------------------------------------
79
80 Conversions between UTF32, UTF-16, and UTF-8. Header file.
81
82 Several funtions are included here, forming a complete set of
83 conversions between the three formats. UTF-7 is not included
84 here, but is handled in a separate source file.
85
86 Each of these routines takes pointers to input buffers and output
87 buffers. The input buffers are const.
88
89 Each routine converts the text between *sourceStart and sourceEnd,
90 putting the result into the buffer between *targetStart and
91 targetEnd. Note: the end pointers are *after* the last item: e.g.
92 *(sourceEnd - 1) is the last item.
93
94 !!! NOTE: The source and end pointers must be aligned properly !!!
95
96 The return result indicates whether the conversion was successful,
97 and if not, whether the problem was in the source or target buffers.
98 (Only the first encountered problem is indicated.)
99
100 After the conversion, *sourceStart and *targetStart are both
101 updated to point to the end of last text successfully converted in
102 the respective buffers.
103
104 Input parameters:
105 sourceStart - pointer to a pointer to the source buffer.
106 The contents of this are modified on return so that
107 it points at the next thing to be converted.
108 targetStart - similarly, pointer to pointer to the target buffer.
109 sourceEnd, targetEnd - respectively pointers to the ends of the
110 two buffers, for overflow checking only.
111
112 These conversion functions take a ConversionFlags argument. When this
113 flag is set to strict, both irregular sequences and isolated surrogates
114 will cause an error. When the flag is set to lenient, both irregular
115 sequences and isolated surrogates are converted.
116
117 Whether the flag is strict or lenient, all illegal sequences will cause
118 an error return. This includes sequences such as: <F4 90 80 80>, <C0 80>,
119 or <A0> in UTF-8, and values above 0x10FFFF in UTF-32. Conformant code
120 must check for illegal sequences.
121
122 When the flag is set to lenient, characters over 0x10FFFF are converted
123 to the replacement character; otherwise (when the flag is set to strict)
124 they constitute an error.
125
126 Output parameters:
127 The value "sourceIllegal" is returned from some routines if the input
128 sequence is malformed. When "sourceIllegal" is returned, the source
129 value will point to the illegal value that caused the problem. E.g.,
130 in UTF-8 when a sequence is malformed, it points to the start of the
131 malformed sequence.
132
133 Author: Mark E. Davis, 1994.
134 Rev History: Rick McGowan, fixes & updates May 2001.
135 Fixes & updates, Sept 2001.
136
137------------------------------------------------------------------------ */

141/* ---------------------------------------------------------------------
142 The following 4 definitions are compiler-specific.
143 The C standard does not guarantee that wchar_t has at least
144 16 bits, so wchar_t is no less portable than unsigned short!
145 All should be unsigned values to avoid sign extension during
146 bit mask & shift operations.
147------------------------------------------------------------------------ */
