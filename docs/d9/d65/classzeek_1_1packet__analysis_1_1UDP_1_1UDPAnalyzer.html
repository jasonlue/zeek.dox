Upon seeing the first packet of a connection, checks whether we want to analyze it (e.g. we may not want to look at partial connections) and, if yes, whether we should flip the roles of originator and responder based on known ports and such.

Parameters
src_portThe source port of the connection.
dst_portThe destination port of the connection.
dataThe payload data for the packet being processed.
flip_rolesReturn value if the roles should be flipped.
Returns
True if the connection is wanted. False otherwise.
