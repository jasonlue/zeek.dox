3/***
4 * This file contains functions to generate hashes using keyed hash functions.
5 * Keyed hash functions make it difficult/impossible to find information about the
6 * output of a hash when the key is unknown to the attacker. This fact holds, even
7 * when the input value is known.
8 *
9 * We use these kinds of hashes heavily internally - e.g. for scriptland hash generation.
10 * It is important that these hashes are not easily guessable to prevent complexity attacks.
11 *
12 * The HashKey class is the actual class that is used to generate Hash keys that are used
13 * internally, e.g. for lookups in hash-tables and connection ID generation.
14 *
15 * This means that the hashes created by most functions in this file will be different each run,
16 * unless a seed file is used. There are a few functions that create hashes that are static over
17 * runs and use an installation-wide seed value; these are specifically called out.
18 */
19
