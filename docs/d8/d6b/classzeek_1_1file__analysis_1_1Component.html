Detailed Description

Component description for plugins providing file analyzers.

A plugin can provide a specific file analyzer by registering this analyzer component, describing the analyzer.

Constructor.

Parameters
nameThe name of the provided analyzer. This name is used across the system to identify the analyzer, e.g., when calling file_analysis::Manager::InstantiateAnalyzer with a name.
factoryA factory function to instantiate instances of the analyzer's class, which must be derived directly or indirectly from file_analysis::Analyzer. This is typically a static Instatiate() method inside the class that just allocates and returns a new instance.
subtypeA subtype associated with this component that further distinguishes it. The subtype will be integrated into the Tag that the manager associates with this analyzer, and analyzer instances can accordingly access it via Tag(). If not used, leave at zero.
enabledIf false the analyzer starts out as disabled and hence won't be used. It can still be enabled later via the manager, including from script-land.
