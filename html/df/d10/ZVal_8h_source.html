44 // Note that a ZVal by itself is ambiguous: it doesn't track its type.
45 // This makes them consume less memory and cheaper to copy. It does
46 // however require a separate way to determine the type. Generally
47 // this is doable using surrounding context, or can be statically
48 // determined in the case of optimization/compilation.
49 //
50 // An alternative would be to use std::variant, but it will be larger
51 // due to needing to track the variant type, and it won't allow access
52 // to the managed_val member, which both simplifies memory management
53 // and is also required for sharing of ZAM frame slots.
54
